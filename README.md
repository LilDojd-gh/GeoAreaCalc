# GeoAreaCalc: Intelligent Satellite Area Calculator

Интеллектуальная система компьютерного зрения для семантической сегментации строений и автоматического расчета площади застройки по спутниковым снимкам и данным с БПЛА.

## О проекте
В классических задачах анализа спутниковых снимков перевод пикселей в квадратные метры требует наличия метаданных (GSD — Ground Sample Distance), которые часто отсутствуют в обычных изображениях (.jpg, .png). 

**GeoAreaCalc** решает эту проблему с помощью ансамбля из двух кастомных нейросетей:
1. **Базовая задача (Семантическая сегментация):** Модель **U-Net** выделяет точные контуры зданий. Для борьбы с дисбалансом классов применена гибридная функция потерь `BCE + Dice Loss`.
2. **Альтернативная задача (Детекция объектов):** Модель **Faster R-CNN** находит на снимке автомобили. Используя медианную длину найденных машин в качестве физического эталона ($\approx 4.5$ м), система динамически вычисляет GSD и переводит площадь сегментированных крыш из пикселей в реальные $m^2$.

## 📊 Данные (RescueNet Dataset)
В качестве основы для обучения используется высокодетализированный датасет аэрофотосъемки **RescueNet**. 
* **Сегментация:** Исходные многоклассовые маски (целые, поврежденные и разрушенные здания — классы 2, 3, 4, 5) автоматически объединяются в единый бинарный класс "Строение".
* **Детекция:** Автомобили (класс 6) извлекаются из масок RescueNet и конвертируются в Bounding Boxes формата YOLO для обучения Faster R-CNN.

## Структура репозитория

```text
satellite_vision_project/
├── data/                       # [GitIgnored] Датасеты RescueNet (картинки, маски, yolo-txt)
├── weights/                    # [GitIgnored] Папка для весов (на GitHub пушатся только *best.pth)
├── notebooks/
│   └── research.ipynb          # Аналитика (EDA), расчет дисбаланса, метрики, динамика лоссов
├── src/                        # Исходный код пакета
│   ├── __init__.py             # Инициализация модуля
│   ├── dataset.py              # Классы загрузчиков с аугментациями (Albumentations)
│   ├── models.py               # Архитектуры U-Net и кастомная "голова" Faster R-CNN
│   ├── losses.py               # Комбинированная функция потерь BCEDiceLoss
│   └── utils.py                # Расчет IoU, Dice, GSD и площади строений (OpenCV CCA)
├── train_segmentation.py       # Custom Training Loop для обучения U-Net
├── train_detection.py          # Custom Training Loop для обучения Faster R-CNN
├── app.py                      # Интерактивный веб-интерфейс (Streamlit)
├── requirements.txt            # Зависимости проекта
└── README.md                   # Описание проекта
Установка и запуск (Деплой)
Система протестирована на Python 3.10+. Для развертывания проекта на локальной машине выполните следующие шаги:

1. Клонирование репозитория
Bash
git clone [https://github.com/LilDojd-gh/GeoAreaCalc.git]
cd GeoAreaCalc
2. Создание и активация виртуального окружения
Bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS/Linux
python3 -m venv venv
source venv/bin/activate
3. Установка зависимостей
Все необходимые библиотеки, включая PyTorch, OpenCV и Streamlit, установятся автоматически:

Bash
pip install -r requirements.txt
4. Запуск веб-интерфейса
Убедитесь, что обученные веса (unet_best.pth и detector_best.pth) находятся в папке weights/. Запустите приложение командой:

Bash
streamlit run app.py
Интерфейс будет доступен в браузере по адресу: http://localhost:8501

    Использование
Откройте веб-приложение в браузере.

С помощью ползунков в боковой панели настройте пороги уверенности (Confidence Threshold) для обеих нейросетей, если это необходимо.

Загрузите снимок местности в формате JPG/PNG.

Система выведет два изображения: размеченные машины с рассчитанным масштабом и пообъектную карту строений.

В нижней части экрана сгенерируется аналитический отчет с общей площадью застройки и детализированной таблицей площади каждого отдельного здания.

    Стек технологий
PyTorch & Torchvision — создание архитектур, кастомные циклы обучения (без готовых пайплайнов).

OpenCV — морфологическая обработка масок, метод связных компонент (CCA).

Albumentations — оптимизированный пайплайн аугментаций тензоров.

Streamlit — разработка интерактивного веб-сервиса.

Jupyter — исследовательский анализ данных (EDA).
